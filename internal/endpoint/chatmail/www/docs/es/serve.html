Guía de personalización del servidor (HTML propio)

Puede cambiar el aspecto de todas las páginas web del servidor sin modificar el código fuente ni recompilar.

Paso 1 — Extraer los archivos originales

Primero extraiga los archivos por defecto en una carpeta. Recomendamos /var/lib/maddy/www.

sudo maddy extract-www /var/lib/maddy/www

Paso 2 — Permisos

Para que el servidor pueda leer los archivos, asigne la carpeta al usuario maddy.

sudo chown -R maddy:maddy /var/lib/maddy/www

Paso 3 — Activar en el servidor

Actualice maddy.conf para usar la nueva carpeta.

sudo sed -i '/chatmail.*{/a\ www_dir /var/lib/maddy/www' /etc/maddy/maddy.conf

Paso 4 — Aplicar cambios

Reinicie el servicio para que surtan efecto los cambios.

sudo systemctl restart maddy
Importante: después de estos pasos puede editar los archivos en /var/lib/maddy/www con un editor. Los cambios se aplican al recargar la página.

Nota sobre el registro JIT

Si personaliza la página principal (index.html), tenga en cuenta que la lógica de registro depende del estado de JIT:

  • Si JIT está activado, las credenciales se generan al azar en el navegador.
  • Si JIT está desactivado (pero el registro está abierto), el navegador usa la API /new para crear cuentas.

Recomendamos revisar el JavaScript de la función generateAccount en el archivo por defecto.

Volver al predeterminado

Para usar de nuevo los archivos integrados en el binario:

sudo sed -i '/www_dir/d' /etc/maddy/maddy.conf sudo systemctl restart maddy
