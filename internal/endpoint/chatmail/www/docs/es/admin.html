Documentación del administrador del sistema

Esta guía explica cómo gestionar cuotas de usuario y ver estadísticas del servidor con los comandos maddy.

💡 Consejo: todas estas operaciones también están disponibles en la interfaz web de administración en /admin/ — no hace falta usar la línea de órdenes.

Gestión de cuotas

Puede administrar las cuotas de almacenamiento con los subcomandos maddy imap-acct quota.

Ver la cuota de un usuario

Muestra el espacio usado y el límite máximo de un usuario concreto.

maddy imap-acct quota get usuario@example.com

Fijar cuota de usuario

Establece un límite personalizado (p. ej. 1GB, 500MB o 10KB).

maddy imap-acct quota set usuario@example.com 1GB

Restablecer cuota de usuario

Quita la cuota personalizada y vuelve al valor por defecto del sistema. Se conserva la fecha de creación de la cuenta.

maddy imap-acct quota reset usuario@example.com

Cuota por defecto global

Define el límite por defecto para todos los usuarios sin cuota propia. El valor se guarda en la base de datos y tiene prioridad sobre maddy.conf.

maddy imap-acct quota set-default 2GB

Listar todas las cuotas

Muestra todos los usuarios con espacio usado y límites. Indica si la cuota es personalizada o heredada del valor por defecto.

maddy imap-acct quota list

Mantenimiento y limpieza

Órdenes para aliviar la carga del servidor y liberar espacio eliminando datos de usuario.

Vaciar la cola de envío

Elimina los mensajes pendientes en la cola de envío relacionados con un usuario (remitente o destinatario). Útil si hay atascos por spam o mensajes grandes.

maddy queue purge usuario@example.com

Borrar correo IMAP de un usuario

Elimina todos los correos de un usuario en todas las carpetas (Bandeja de entrada, Enviados, etc.) sin borrar la cuenta ni la cuota. Útil para liberar disco rápidamente.

maddy imap-acct purge-msgs usuario@example.com

Borrar todo el correo de todos los usuarios

Elimina el correo de todo el servidor manteniendo cuentas, contraseñas y cuotas.

maddy imap-acct purge-all

Borrar todos los mensajes leídos

Elimina los mensajes marcados como leídos (Seen) en todo el servidor. Los no leídos se conservan.

maddy imap-acct purge-read [--yes]

Podar mensajes no leídos antiguos

Elimina mensajes no leídos más antiguos que la duración indicada (p. ej. 30 días).

maddy imap-acct prune-unread 720h [--yes]

Gestión del registro

Puede abrir o cerrar el registro de nuevos usuarios. Si está cerrado, solo pueden entrar quienes ya tienen cuenta.

Abrir registro

maddy creds registration open

Cerrar registro

maddy creds registration close

Ver estado actual

maddy creds registration status

Registro Just-In-Time (JIT)

JIT permite crear cuentas automáticamente en el primer inicio de sesión. Se gestiona aparte del registro general.

Activar JIT

Con JIT activo, las cuentas se crean en el primer login o al recibir el primer correo.

maddy creds jit enable

Desactivar JIT

Sin JIT, las cuentas solo se crean vía API (interfaz web) y se rechaza el login con credenciales no registradas.

maddy creds jit disable

Ver estado de JIT

maddy creds jit status

Comandos de cuentas (maddy accounts)

Estos subcomandos leen su maddy.conf y acceden a las bases de datos en disco directamente (no hace falta parar el servidor). Use --cfg-block y --storage-cfg-block si sus bloques de autenticación o IMAP no son local_authdb / local_mailboxes.

Estado e información por usuario

maddy accounts status maddy accounts info usuario@example.com

Crear acceso e IMAP

Crea credenciales y buzón IMAP (equivalente al usuario aleatorio del panel de administración).

maddy accounts create usuario@example.com maddy accounts create-random [--json-only]

Eliminar o vetar

Borra la fila de contraseña, el almacenamiento de correo y bloquea la dirección (como «eliminar» en el admin). ban admite un motivo opcional como segundo argumento.

maddy accounts delete usuario@example.com [--yes] maddy accounts ban usuario@example.com ["motivo"] [--yes]

Tras cambios por CLI con el servidor en marcha

El demonio mantiene copias en memoria de credenciales y cuotas por rendimiento. Si modificó la base en vivo con la CLI, invoque una vez POST /admin/cache/reload (mismo token de admin que el resto de la API) para que el proceso vuelva a leer las tablas. Véase la documentación de la Admin API para el sobre JSON exacto.

Servidor de llamadas (TURN/STUN)

Puede activar o desactivar TURN para todos los usuarios. Si está desactivado, la página de información no muestra datos de llamadas.

Activar servidor de llamadas

maddy creds turn on

Desactivar servidor de llamadas

maddy creds turn off

Ver estado

maddy creds turn status

Registros (política sin logs)

Puede activar o desactivar el registro de actividad del servidor. Si está desactivado, no se guarda información de uso.

Desactivar logging

Por privacidad y para no almacenar actividad de usuarios.

maddy creds log disable

Activar logging

maddy creds log enable

Ver estado

maddy creds log status

Idioma del sitio web

Puede cambiar el idioma de la interfaz. Idiomas admitidos: inglés (en), farsi (fa), ruso (ru), español (es). Los cambios son inmediatos sin reiniciar.

Ver idioma actual

maddy language

Fijar idioma

Cambia el idioma del sitio.

maddy language set es

Volver al valor por defecto

Quita la sobrescritura en la base de datos y usa el idioma de maddy.conf.

maddy language reset

Puertos y parámetros de servicio

Puertos y ajustes (hostname, secret, IP del relé, etc.) se gestionan por la Admin API. Se guardan en la base de datos y sustituyen a los valores del archivo de configuración.

Parámetros gestionables

  • Puertos: SMTP, Submission, IMAP, TURN, Dovecot, Iroh, Shadowsocks
  • Configuración: hostname SMTP, realm/secret/IP/TTL de TURN, URL del relé Iroh, cifrado y contraseña de Shadowsocks

Use la Admin API. Los extremos están en las secciones «Port Settings» y «Configuration Settings» de la documentación de la API.

Modo sigiloso

Madmail puede operar en modo sigiloso: solo quedan expuestos los puertos imprescindibles.

Puertos esenciales:

  • 80 y 443 (HTTP/HTTPS): registro e interfaz web.
  • 8388 (Shadowsocks): conexiones seguras y rápidas desde la app Delta Chat.

Otros puertos (25, 143, 587, 993) pueden cerrarse al tráfico público con el cortafuegos. Los usuarios enlazan los servicios a través del proxy Shadowsocks en la app, reduciendo la huella frente a escaneos.

Desactivar la Admin API

La Admin API está activada por defecto. Para desactivarla por completo en maddy.conf, en el bloque chatmail:

Ajuste en maddy.conf

Ponga admin_token en disabled:

chatmail tls://0.0.0.0:443 { # ... otros ajustes ... # Desactivar por completo la Admin API: admin_token disabled }

Guarde el archivo y reinicie maddy. Se deshabilitan el panel web y la Admin API.

Caché DNS y reescritura de direcciones

El sistema interno de caché DNS permite redirigir el enrutado del correo saliente sin cambiar el DNS del sistema operativo. Las reglas están en la base del programa (almacenamiento). Los cambios son inmediatos sin reiniciar.

Ver todas las reescrituras

Muestra las reglas DNS guardadas en la base.

maddy dns-cache list

Crear o editar reescritura

Define una regla. Si ya existe para esa dirección, se actualiza.

maddy dns-cache set CLAVE_BÚSQUEDA HOST_DESTINO [COMENTARIO]

Ejemplos:

# Enrutar dominio a servidor interno maddy dns-cache set nine.testrun.org 10.0.0.5 "Entorno de pruebas" # Reescribir IP maddy dns-cache set 1.1.1.1 2.2.2.2 "Redirigir IP" # Migración a nuevo servidor maddy dns-cache set example.com new-mx.example.com "Migración"

Ver una reescritura concreta

Muestra origen, destino, comentarios y fecha de creación.

maddy dns-cache get nine.testrun.org

Borrar reescritura

Elimina la regla. El correo vuelve a usar el DNS normal del SO.

maddy dns-cache remove nine.testrun.org

Cómo funciona

  • Prioridad: primero la base interna; si no hay regla, DNS del SO.
  • Sin distinguir mayúsculas: las direcciones coinciden sin importar capitalización.
  • Dominios e IP: se pueden reescribir nombres (p. ej. example.com) e IP (p. ej. 1.1.1.1).
  • Sin reinicio: afecta de inmediato al correo nuevo.
  • Reserva: si la base no está disponible, se usa DNS normal.

Estadísticas del servidor

Resumen global

Muestra el almacenamiento total usado y el número de cuentas activas.

maddy imap-acct stat

Monitorización de conexiones

El comando maddy status muestra el estado en tiempo real: conexiones IMAP, relés TURN (llamadas), Shadowsocks, usuarios registrados y tiempo activo.

Ver conexiones activas

Resumen con conexiones, usuarios, arranque y uptime.

maddy status

Ejemplo de salida:

IMAP            connections: 27     unique IPs: 7
TURN            relays: 1
Shadowsocks     connections: 0      unique IPs: 0

Registered users:   42
Boot time:          2026-02-16 20:56:19 (up 5m 24s)

Email servers seen (since last restart):
  Connection IPs:   1
  Domain servers:   1
  IP servers:       0

Detalle por puerto

Use la opción --details (o -d) para desglosar por puerto.

maddy status --details

Ejemplo de salida:

PORT   PROTO   TYPE              CONNECTIONS   UNIQUE IPs
993    tcp     IMAP TLS          25            7
143    tcp     IMAP              0             0
3478   udp     TURN UDP          1 relays      -
3478   tcp     TURN TCP          0             0
8388   tcp     Shadowsocks       0             0
443    tcp     ALPN (chatmail)   3             1

Significado de los campos

  • IMAP: conexiones activas de usuarios al servidor de correo.
  • Relés TURN: relés de voz/vídeo activos; cada uno suele ser una llamada.
  • Shadowsocks: conexiones activas a través del proxy.
  • Hora de arranque: inicio del servidor y duración del uptime.
  • Servidores de correo vistos: servidores únicos que han enviado correo desde el último reinicio; los datos se guardan con resúmenes criptográficos sin exponer información privada.
