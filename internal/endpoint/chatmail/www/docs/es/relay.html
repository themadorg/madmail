🌐 Dominio y configuración TLS

Paso 1: registros DNS

Configure los siguientes registros DNS en el panel de su dominio:

# Registro A — el dominio apunta a la IP del servidor
chat.example.org. A 203.0.113.50
Nota: Tras crear los registros, la propagación puede tardar entre minutos y 24 horas. Para comprobar: dig +short chat.example.org A

Paso 2: modo TLS

Madmail admite cuatro modos TLS. La tabla resume las diferencias:

Modo Desafío API DNS Puerto 80 Caso de uso
autocert HTTP-01 No Necesario Servidor en producción
acme DNS-01 Sí No Servidor tras cortafuegos
file — — — Certificado existente
self_signed — — — Pruebas y desarrollo

🏆 modo autocert Recomendado

Instalación (maddy install)
sudo maddy install \
  --domain chat.example.org \
  --tls-mode autocert \
  --acme-email admin@example.org \
  --enable-chatmail \
  --non-interactive

Cómo funciona:

  1. Madmail sirve HTTP en el puerto 80 para el desafío ACME
  2. Let's Encrypt comprueba el dominio contactando el puerto 80
  3. Se emite un certificado (90 días, renovación automática)
  4. Los certificados se guardan en /var/lib/maddy/autocert/

En el archivo de configuración (maddy.conf):

maddy.conf
tls {
    loader autocert {
        hostname chat.example.org
        email admin@example.org
        cache_dir /var/lib/maddy/autocert
        agreed
    }
}

🔐 modo acme (DNS-01) Avanzado

Instalación (maddy install)
sudo maddy install \
  --domain chat.example.org \
  --tls-mode acme \
  --acme-email admin@example.org \
  --acme-dns-provider cloudflare \
  --acme-dns-token "YOUR_CLOUDFLARE_TOKEN" \
  --enable-chatmail \
  --non-interactive

Proveedores DNS admitidos: cloudflare, gandi, digitalocean, vultr, hetzner, route53

📄 modo file (certificado existente)

Instalación (maddy install)
sudo maddy install \
  --domain chat.example.org \
  --tls-mode file \
  --cert-path /etc/letsencrypt/live/chat.example.org/fullchain.pem \
  --key-path /etc/letsencrypt/live/chat.example.org/privkey.pem \
  --enable-chatmail

Formato en el archivo de configuración:

tls file /path/to/fullchain.pem /path/to/privkey.pem
Renovación: configure un hook de certbot para reiniciar maddy tras renovar:
systemctl restart maddy

🧪 modo self_signed (pruebas)

Instalación (maddy install)
sudo maddy install \
  --simple --ip YOUR_SERVER_IP \
  --tls-mode self_signed \
  --turn-off-tls

Los certificados se guardan en /var/lib/maddy/certs/ y se reutilizan al reiniciar.

Paso 3: verificar el certificado

Tras la configuración, compruebe el certificado con:

echo | openssl s_client -connect chat.example.org:443 \ -servername chat.example.org 2>&1 | \ openssl x509 -noout -subject -issuer -dates

Salida esperada:

subject=CN=chat.example.org issuer=C=US, O=Let's Encrypt, CN=E7 notBefore=Feb 21 15:41:47 2026 GMT notAfter=May 22 15:41:46 2026 GMT

Cambiar de modo TLS

De self_signed a autocert

Eliminar certificados antiguos
sudo rm /var/lib/maddy/certs/fullchain.pem /var/lib/maddy/certs/privkey.pem
Reinstalar con autocert
sudo maddy install \
  --domain chat.example.org \
  --tls-mode autocert \
  --acme-email admin@example.org \
  --enable-chatmail \
  --non-interactive
Reinicio final
sudo systemctl restart maddy

Resolución de problemas

🔑 file: ¿error «Permission denied»?

Ejecute lo siguiente para corregir propietario y permisos:

Corregir permisos
sudo chown root:maddy /path/to/cert.pem /path/to/key.pem
sudo chmod 640 /path/to/cert.pem /path/to/key.pem
🛡️ Notas de seguridad:
  • Las claves privadas suelen tener permisos 600 (solo el propietario)
  • Los certificados suelen tener permisos 640 (propietario y grupo)
  • El directorio de caché de autocert suele tener 700 (solo el propietario)
  • Los certificados de Let's Encrypt duran 90 días y se renuevan unas semanas antes de caducar
