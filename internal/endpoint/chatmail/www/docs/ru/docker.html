🐳 Настройка Docker

Быстрый старт

docker pull ghcr.io/themadorg/madmail:latest

Теги образов (Image Tags)

Тег Описание
latest Последняя стабильная версия из основной ветки (main)
X.Y.Z Конкретная версия (например, 0.18.3)

Метод 1: Простой запуск по IP (SQLite)

docker run -d --name madmail \ -p 25:25 \ -p 143:143 \ -p 465:465 \ -p 587:587 \ -p 993:993 \ -p 80:80 \ -p 443:443 \ -p 1080:1080 \ -v ./maddy.conf:/data/maddy.conf \ -v ./data:/data \ -v ./tls:/data/tls \ -e MADDY_HOSTNAME=IP_ВАШЕГО_СЕРВЕРА \ -e MADDY_DOMAIN="[IP_ВАШЕГО_СЕРВЕРА]" \ -e MADDY_PUBLIC_IP=IP_ВАШЕГО_СЕРВЕРА \ ghcr.io/themadorg/madmail:latest

Вам понадобятся файлы docker-compose.yml и maddy.conf. В этом режиме автоматически генерируются самоподписанные TLS-сертификаты (self-signed).

docker-compose.yml
services:
  madmail:
    image: ghcr.io/themadorg/madmail:latest
    restart: always
    ports:
      - "25:25"      # SMTP
      - "143:143"    # IMAP (STARTTLS)
      - "465:465"    # Submission (TLS/SSL)
      - "587:587"    # Submission (STARTTLS)
      - "993:993"    # IMAPS (TLS/SSL)
      - "80:80"      # HTTP
      - "443:443"    # HTTPS
      - "1080:1080"  # Shadowsocks
    volumes:
      - ./maddy.conf:/data/maddy.conf
      - ./data:/data
      - ./tls:/data/tls
    environment:
      - MADDY_HOSTNAME=IP_ВАШЕГО_СЕРВЕРА
      - MADDY_DOMAIN=[IP_ВАШЕГО_СЕРВЕРА]
      - MADDY_PUBLIC_IP=IP_ВАШЕГО_СЕРВЕРА
      - MADDY_SQLITE_UNSAFE_SYNC_OFF=1
Примечание: При использовании IP вместо домена необходимо обернуть его в квадратные скобки в MADDY_DOMAIN: [203.0.113.50]

Метод 2: С использованием PostgreSQL

Ошибка database is locked в SQLite решается переходом на PostgreSQL. Также становится доступным pub/sub для обновления IMAP в реальном времени.

docker-compose.yml
services:
  madmail:
    image: ghcr.io/themadorg/madmail:latest
    restart: always
    ports:
      - "25:25"
      - "143:143"
      - "465:465"
      - "587:587"
      - "993:993"
      - "80:80"
      - "443:443"
      - "1080:1080"
    volumes:
      - ./maddy.conf:/data/maddy.conf
      - ./data:/data
      - ./tls:/data/tls
    environment:
      - MADDY_HOSTNAME=IP_ВАШЕГО_СЕРВЕРА
      - MADDY_DOMAIN=[IP_ВАШЕГО_СЕРВЕРА]
      - MADDY_PUBLIC_IP=IP_ВАШЕГО_СЕРВЕРА
      - POSTGRES_USER=madmail
      - POSTGRES_PASSWORD=madmail_pass
      - POSTGRES_DB=madmail
    depends_on:
      - db

  db:
    image: postgres:alpine
    restart: always
    environment:
      - POSTGRES_USER=madmail
      - POSTGRES_PASSWORD=madmail_pass
      - POSTGRES_DB=madmail
    volumes:
      - ./pgdata:/var/lib/postgresql/data

Метод 3: Домен с автоматическим TLS (Let's Encrypt)

⚠️ Предварительные требования:
  • Реальный домен с DNS-записью типа A, указывающей на ваш сервер
  • Порт 80 открыт и доступен из интернета (для ACME-проверки)
  • Порт 443 открыт (для HTTPS)
docker-compose.yml
services:
  madmail:
    image: ghcr.io/themadorg/madmail:latest
    restart: always
    ports:
      - "25:25"
      - "143:143"
      - "465:465"
      - "587:587"
      - "993:993"
      - "80:80"
      - "443:443"
      - "1080:1080"
    volumes:
      - ./maddy.conf:/data/maddy.conf
      - ./data:/data
    environment:
      - MADDY_HOSTNAME=chat.example.org
      - MADDY_DOMAIN=chat.example.org
      - MADDY_PUBLIC_IP=203.0.113.50
      - MADDY_ACME_EMAIL=admin@example.org
Как это работает:
  1. Madmail запускает HTTP-сервер на порту 80 для ACME-проверки
  2. Let's Encrypt выдает сертификат при первом TLS-подключении
  3. Сертификаты хранятся в /data/autocert/ и продлеваются автоматически
  4. Обычные HTTP-запросы (не ACME) перенаправляются на HTTPS (302)

Compose Build
