🌐 Домен и настройка TLS

Шаг 1: Настройка DNS-записей

Сначала настройте следующие DNS-записи в вашей панели управления доменом:

# Запись типа A — укажите IP-адрес сервера
chat.example.org. A 203.0.113.50
Примечание: Распространение записей DNS может занять от 5 минут до 24 часов. Для мгновенной проверки: dig +short chat.example.org A

Шаг 2: Выберите режим TLS

Madmail поддерживает четыре режима TLS. Различия показаны в таблице ниже:

Режим Метод проверки (Challenge) DNS API Порт 80 Применение
autocert HTTP-01 Нет Обязательно Продакшн-сервер
acme DNS-01 Да Нет За файрволом
file — — — Готовый сертификат
self_signed — — — Разработка и тесты

🏆 Режим autocert Рекомендуется

Установка (Maddy Install)
sudo maddy install \
  --domain chat.example.org \
  --tls-mode autocert \
  --acme-email admin@example.org \
  --enable-chatmail \
  --non-interactive

Как это работает:

  1. Madmail запускает HTTP-сервер на порту 80 для ACME-проверки
  2. Let's Encrypt отправляет запрос на порт 80 для подтверждения владения доменом
  3. Выпускается сертификат (выдается на 90 дней, продлевается автоматически)
  4. Сертификаты хранятся в /var/lib/maddy/autocert/

В файле конфигурации (maddy.conf):

maddy.conf
tls {
    loader autocert {
        hostname chat.example.org
        email admin@example.org
        cache_dir /var/lib/maddy/autocert
        agreed
    }
}

🔐 Режим acme (DNS-01) Продвинутый

Установка (Maddy Install)
sudo maddy install \
  --domain chat.example.org \
  --tls-mode acme \
  --acme-email admin@example.org \
  --acme-dns-provider cloudflare \
  --acme-dns-token "ВАШ_ТОКЕН_CLOUDFLARE" \
  --enable-chatmail \
  --non-interactive

Поддерживаемые DNS-провайдеры: cloudflare, gandi, digitalocean, vultr, hetzner, route53

📄 Режим file (Готовый сертификат)

Установка (Maddy Install)
sudo maddy install \
  --domain chat.example.org \
  --tls-mode file \
  --cert-path /etc/letsencrypt/live/chat.example.org/fullchain.pem \
  --key-path /etc/letsencrypt/live/chat.example.org/privkey.pem \
  --enable-chatmail

Формат файла конфигурации:

tls file /путь/к/fullchain.pem /путь/к/privkey.pem
Продление: Настройте certbot-хук для перезапуска maddy после продления:
systemctl restart maddy

🧪 Режим self_signed (Тестирование)

Установка (Maddy Install)
sudo maddy install \
  --simple --ip ВАШ_IP_СЕРВЕРА \
  --tls-mode self_signed \
  --turn-off-tls

Сертификаты хранятся в /var/lib/maddy/certs/ и используются при последующих запусках.

Шаг 3: Проверка сертификата

После настройки проверьте ваш сертификат следующей командой:

echo | openssl s_client -connect chat.example.org:443 \ -servername chat.example.org 2>&1 | \ openssl x509 -noout -subject -issuer -dates

Пример вывода:

subject=CN=chat.example.org issuer=C=US, O=Let's Encrypt, CN=E7 notBefore=Feb 21 15:41:47 2026 GMT notAfter=May 22 15:41:46 2026 GMT

Смена режима TLS

С self_signed на autocert

Удаление старых сертификатов
sudo rm /var/lib/maddy/certs/fullchain.pem /var/lib/maddy/certs/privkey.pem
Переустановка с autocert
sudo maddy install \
  --domain chat.example.org \
  --tls-mode autocert \
  --acme-email admin@example.org \
  --enable-chatmail \
  --non-interactive
Финальный перезапуск
sudo systemctl restart maddy

Решение проблем

🔑 Ошибка разрешения доступа к файлу?

Выполните следующие команды для исправления прав доступа:

Исправить права доступа
sudo chown root:maddy /path/to/cert.pem /path/to/key.pem
sudo chmod 640 /path/to/cert.pem /path/to/key.pem
🛡️ Заметки о безопасности:
  • Приватные ключи хранятся с правами 600 (только владелец)
  • Сертификаты хранятся с правами 640 (владелец + группа)
  • Директория кэша autocert хранится с правами 700 (только владелец)
  • Сертификаты Let's Encrypt действительны в течение 90 дней и обновляются автоматически примерно за 30 дней до истечения срока
