Настройка базы данных

Madmail теперь поддерживает GORM, что позволяет использовать различные базы данных и автоматическую миграцию (Auto Migration).

Важно: SQLite используется по умолчанию, но для высокой масштабируемости рекомендуется использовать PostgreSQL.

SQLite (по умолчанию)

Самая простая настройка для быстрого развертывания. Файл базы данных хранится по указанному пути.

storage.imapsql local_mail {
    driver sqlite3
    dsn ./madmail.db
}

PostgreSQL

Чтобы использовать PostgreSQL, сначала создайте базу данных на вашем сервере (вы можете использовать docker-compose.yml из этого проекта).

Пример конфигурации в maddy.conf:

storage.imapsql local_mail {
    driver postgres
    dsn "host=localhost user=test password=test dbname=test port=5432 sslmode=disable"
}

Учетные данные по умолчанию в Docker:

  • Пользователь: test
  • Пароль: test
  • База данных: test

MySQL

Чтобы использовать MySQL, используйте следующий формат DSN:

storage.imapsql local_mail {
    dsn "user:password@tcp(localhost:3306)/dbname?charset=utf8mb4&parseTime=True&loc=Local"
}

Миграция

Сервис автоматически создает или обновляет необходимые таблицы при первом запуске. Нет необходимости выполнять SQL-команды вручную для создания таблиц quotas или contacts.
